# Lead-derived deals: leads as inquirers, not primary contacts — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop treating buyer-side leads as the primary contact on auto-created seller-rep deals. Make `Deal.contactId` nullable, drop the lead from the create path, surface them as inquirers, and backfill existing wrong deals.

**Architecture:** One Prisma migration (column nullable + FK `SetNull`); one targeted change to `upsertDealForLead` (no `contactId` on create, no `syncContactRoleFromDeals`); one null-safe SQL tweak to `findRelatedLeadsForDeal`; one UI null-handling pass on the deal detail page; one idempotent backfill script that nulls `contact_id` on lead-derived deals whose contact is itself a lead.

**Tech Stack:** Next.js 15 (App Router), Prisma 6 + Postgres (Supabase), TypeScript, Vitest, pnpm.

**Spec:** [docs/superpowers/specs/2026-04-30-lead-derived-deal-contact-design.md](../specs/2026-04-30-lead-derived-deal-contact-design.md)

**Repo conventions to know before starting:**
- All app code lives under `full-kit/`. Run all commands from `full-kit/` unless noted.
- Database is real Supabase Postgres. Read `CLAUDE.md` at the repo root before any Prisma command — there are hard rules about `--shadow-database-url`.
- Always source env first: `cd full-kit && set -a && source .env.local && set +a`.
- Type check: `pnpm exec tsc --noEmit --pretty false`. Tests: `pnpm test` (vitest run).
- Existing tests use Vitest with `vi.mock("@/lib/prisma", …)`. See [lead-to-deal.test.ts](../../../full-kit/src/lib/deals/lead-to-deal.test.ts) for the pattern this plan extends.

- [ ] **Step 3: Update the SQL to be null-safe**

Open [full-kit/src/lib/deals/related-leads.ts](../../../full-kit/src/lib/deals/related-leads.ts). Find:

```sql
    FROM lead_arm la
    INNER JOIN contacts c ON c.id = la.contact_id
    WHERE la.rn = 1
      AND c.archived_at IS NULL
      AND c.lead_source IS NOT NULL
      AND c.id <> ${deal.contactId}
```

Replace with:

```sql
    FROM lead_arm la
    INNER JOIN contacts c ON c.id = la.contact_id
    WHERE la.rn = 1
      AND c.archived_at IS NULL
      AND c.lead_source IS NOT NULL
      AND (${deal.contactId}::text IS NULL OR c.id <> ${deal.contactId})
```

The `::text` cast is required because Postgres can't infer a type for a bare `NULL` literal; explicitly casting tells it this parameter is a text comparand. The `OR` short-circuits when the parameter is NULL, including every contact in the result set.

## Task 5: Backfill script — null out lead-as-contact on existing deals

**Files:**

- [ ] **Step 1: Write the script**

Create `full-kit/scripts/backfill-lead-derived-deal-contacts.mjs` with:

```javascript
// One-off backfill: detach buyer-side leads from being the "primary contact"
// of lead-derived seller-rep deals.
//
// Before this fix, upsertDealForLead set the inquirer (a buyer-side lead)
// as Deal.contactId on auto-created seller-rep deals, even though the
// lead is conceptually an inquirer and the deal's true primary contact
// should be the listing client (which we don't know at inquiry time).
//
// This script finds every Deal where:
//   - dealSource = 'lead_derived', AND
//   - the joined Contact has a non-null lead_source
//     (i.e. the contact is itself a lead, not a manually-assigned
//     listing client whose contact was created without a lead_source).
// For each, it sets contact_id = NULL and re-runs the role-sync logic
// for the formerly-attached contact so their clientType recomputes.
//
// Safe to run multiple times — once a deal's contact_id is null it
// won't match the WHERE again; once a contact's clientType matches
// their deal portfolio the recompute is a no-op.
//
// Run with:
//   cd full-kit
//   set -a && source .env.local && set +a
//   node scripts/backfill-lead-derived-deal-contacts.mjs

import { PrismaClient } from "@prisma/client"

const db = new PrismaClient()

// Mirror of src/lib/contacts/sync-contact-role.ts::computeRole.
// Duplicated (not imported) so this script has no TS build dependency,
// matching the pattern of backfill-contact-client-type.mjs.
function computeRole(deals) {
  if (deals.length === 0) return null
  const hasActiveBuyerSide = deals.some(
    (d) =>
      (d.dealType === "buyer_rep" || d.dealType === "tenant_rep") &&
      d.stage !== "closed"
  )
  if (hasActiveBuyerSide) return "active_buyer_rep_client"
  const hasActiveListing = deals.some(
    (d) => d.dealType === "seller_rep" && d.stage !== "closed"
  )
  if (hasActiveListing) return "active_listing_client"
  return "past_client"
}

async function syncOne(contactId) {
  const contact = await db.contact.findUnique({
    where: { id: contactId },
    select: { id: true, clientType: true, archivedAt: true },
  })
  if (!contact || contact.archivedAt) {
    return { changed: false }
  }
  const deals = await db.deal.findMany({
    where: { contactId, archivedAt: null },
    select: { dealType: true, stage: true, outcome: true },
  })
  const next = computeRole(deals)
  if (next === contact.clientType) {
    return { changed: false }
  }
  await db.contact.update({
    where: { id: contactId },
    data: { clientType: next },
  })
  return { changed: true, from: contact.clientType, to: next }
}

async function main() {
  // 1. Find candidate deals to scrub. Note: the Prisma client uses
  //    camelCase field names (leadSource), not snake_case (lead_source) —
  //    the @map directive in the schema only affects the underlying SQL
  //    column, not the JS property name.
  const deals = await db.deal.findMany({
    where: {
      dealSource: "lead_derived",
      contactId: { not: null },
      contact: { leadSource: { not: null } },
    },
    select: {
      id: true,
      contactId: true,
      propertyAddress: true,
      contact: { select: { id: true, name: true, leadSource: true } },
    },
  })

  console.log(`Found ${deals.length} lead-derived deals with a lead as primary contact`)

  if (deals.length === 0) {
    console.log("Nothing to backfill — exiting.")
    return
  }

  // 2. Log every row before we touch it. Recoverable from logs if a
  //    surprise lurks.
  for (const d of deals) {
    console.log(
      `  deal=${d.id} property="${d.propertyAddress ?? "-"}" ` +
        `contact=${d.contact?.id} (${d.contact?.name}, lead_source=${d.contact?.leadSource})`
    )
  }

  // 3. Null out contact_id, distinct over contactId so we can re-sync
  //    each affected contact only once.
  const affectedContactIds = new Set(deals.map((d) => d.contactId).filter(Boolean))
  const updateRes = await db.deal.updateMany({
    where: { id: { in: deals.map((d) => d.id) } },
    data: { contactId: null },
  })
  console.log(`Nulled contact_id on ${updateRes.count} deals`)

  // 4. Recompute clientType for each affected contact.
  let resyncedChanged = 0
  for (const cid of affectedContactIds) {
    const r = await syncOne(cid)
    if (r.changed) {
      resyncedChanged++
      console.log(`  contact=${cid} clientType ${r.from ?? "null"} → ${r.to ?? "null"}`)
    }
  }
  console.log(
    `Re-synced ${affectedContactIds.size} contacts (${resyncedChanged} clientType changes)`
  )
}

main()
  .catch((e) => {
    console.error(e)
    process.exit(1)
  })
  .finally(async () => {
    await db.$disconnect()
  })
```
